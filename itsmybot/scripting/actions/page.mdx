# Actions
The actions are the main part of the bot. They are used to perform actions when a trigger is met. You can find more information about how to use them [here](/itsmybot/scripting/).

---
# All Actions

## addInviteBonus
Add bonus invites to a user. Requires the [Invite Traker](/itsmybot/addons/invites/) addon.

<CodeGroup>
```yaml {{ title: 'Usage' }}
id: addInviteBonus
args:
  amount: # Amount of bonus invites
```
```yaml {{ title: 'Example' }}
actions:
  - id: addInviteBonus
    args:
      amount: 5 # Amount of bonus invites
```
</CodeGroup>

## randomAction
Run a random action from a list of actions. The actions can be defined in the `actions` array. If the `actions` array is empty, the action will do nothing.

<CodeGroup>
```yaml {{ title: 'Usage' }}
id: randomAction
args:
  actions: # Array of actions to run
```
```yaml {{ title: 'Example' }}
actions:
  - id: randomAction
    args:
      actions:
        - id: sendMessage
          args:
            content: "Hello there !"
        - id: addReaction
          args:
            value: "✅"
```
</CodeGroup>

## sendRequest
Send a http request. New placeholders will be added when you trigger an action after this action. With the format `%data_<object>%`. If you want a value that is inside an object, you can add the path with underscore `_`. For example, if you want the value of `user.id`, you can use `%data_user_id%`.
***Can triggers actions**
<CodeGroup>
```yaml {{ title: 'Usage' }}
- id: sendRequest
  args:
    value: 'https://api.example.com' # URL
    method: 'POST' # Method, optional. Default is GET
    headers: # Optional
      Content-Type: 'application/json' # Headers
    body: # Optional
      key: 'value' # Body
```
```yaml {{ title: 'Example' }}
- id: sendRequest
  args:
    value: 'https://api.mcstatus.io/v2/status/java/mc.hypixel.net:25565' # URL
    actions:
      - id: sendMessage
        args:
          content: "Players %data_players_online%/%data_players_max% | Version %data_version_name_clean%"
```
</CodeGroup>
