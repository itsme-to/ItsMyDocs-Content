# Actions
The actions are the main part of the bot. They are used to perform actions when a trigger is met. You can find more information about how to use them [here](/itsmybot/scripting/).

## createChannel
Create a channel. 

**Can triggers follow-up actions**
<CodeGroup>
```yaml {{ title: 'Usage' }}
id: createChannel
args:
  value: # Channel name
  channel-type: # Channel type, optional. Default is text
  parent: # Parent channel ID, optional
  description: # Channel description, optional
  permission-overwrites: # Channel permissions, optional
    - id: # Role or user ID
      allow: # Permissions to allow, optional
      deny: # Permissions to deny, optional
```
```yaml {{ title: 'Example' }}
actions:
  - id: createChannel
    args:
      value: "new-channel" # Channel name, support placeholders & list for random names
      channel-type: "text" # Channel type
      parent: "123456789012345678" # Parent channel ID
      description: "This is a new channel" # Channel description, support placeholders & list for random descriptions
      permission-overwrites:
        - id: "123456789012345678" # Role or user ID
          allow: ["VIEW_CHANNEL"] # Permissions to allow
          deny: ["SEND_MESSAGES"] # Permissions to deny
  - id: createChannel
    args:
      value: "simple" # Channel name, support placeholders & list for random names
```
</CodeGroup>

## createThread
Create a thread in a channel. If made in a forum channel, it will create a forum post. If made in a text channel, it will create a thread.

**Can triggers follow-up actions**
<CodeGroup>
```yaml {{ title: 'Usage' }}
id: createThread
args:
  value: # String or array of strings, the thread name
  duration: # Thread auto archive duration in seconds, optional
  private: # Whether the thread is private or not, optional. Default is false
  tags: # Thread tags ID, optional (Only for forum posts)
  # <message configuration> required for forum posts (Only for forum posts)
```
```yaml {{ title: 'Example' }}
actions:
  - id: createThread
    args:
      value: "Hello there !" # Thread name
  - id: createThread
    args:
      value:
        - "Thread 1" # Thread name
        - "Thread 2" # Thread name
      duration: 3600 # Thread auto archive duration in seconds (1 hour)
      private: true # Whether the thread is private or not
      tags: # Thread tags ID, optional (Only for forum posts)
        - "123456789012345678" # Tag ID
        - "234567890123456789" # Tag ID
      components:
        - type: text-display
          content: "This is a forum post" # Required for forum posts
```
</CodeGroup>

## sendRequest
Send a http request. New placeholders will be added when you trigger an action after this action. With the format `%data_<object>%`. If you want a value that is inside an object, you can add the path with underscore `_`. For example, if you want the value of `user.id`, you can use `%data_user_id%`.
***Can triggers follow-up actions**
<CodeGroup>
```yaml {{ title: 'Usage' }}
- id: sendRequest
  args:
    value: 'https://api.example.com' # URL
    method: 'POST' # Method, optional. Default is GET
    headers: # Optional
      Content-Type: 'application/json' # Headers
    body: # Optional
      key: 'value' # Body
```
```yaml {{ title: 'Example' }}
- id: sendRequest
  args:
    value: 'https://api.mcstatus.io/v2/status/java/mc.hypixel.net:25565' # URL
    actions:
      - id: sendMessage
        args:
          content: "Players %data_players_online%/%data_players_max% | Version %data_version_name_clean%"
```
</CodeGroup>
